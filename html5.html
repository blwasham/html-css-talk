HTML 5 Philosophy/Principles

Evolution not revolution. Support existing content.
Solve real problems. Do not reinvent the wheel. Pave the cowpaths.
User is most important person to consider
Does not define appearance, only logical structure
Secure by design. Address security directly in specification
Purpose is to enable structure to be understood by any computer

The DOM

The Document Object Model (DOM) is the data representation of the objects that comprise the structure and content of a document on the web.

The Document Object Model (DOM) is a programming interface for HTML and XML documents. It represents the page so that programs can change the document structure, style, and content. The DOM represents the document as nodes and objects. That way, programming languages can connect to the page.
document.getElementById('dom').style.color = 'red';

Semantic Tags

Early versions of HTML had no globally accepted structural elements. HTML 5 semantics is all about using the appropriate elements or tags to help your site be more transportable and device agnostic. These tags are also very important for communicating the meaning of various sections of your website to screen readers and search engines.

Main Semantic Tags

  • Header - top of document
  • Nav - collection of nav items
  • Section - seperate section of page
  • Article - Main content of the page
  • Aside - Support or secondary content of the page
  • Footer - Footnote

Some Benefits

  • Important for accessiblity, especially with simplifying page navigation
  • Help you deal w/ foreign alphabets
  • Make it easier to create consistent styling

Block-level Elements

A block-level element always starts on a new line and takes up the full width available (stretches out to the left and right as far as it can).

Inline Elements

An inline element does not start on a new line and only takes up as much width as necessary.

Inline Block Elements
