Cascading Style Sheets

  • Stands for Cascading Style Sheets and defind how to display HTML elements
  • Styles were added to HTML 4.0 to help separate HTML content from display layout
  • External Style Sheets can save you a lot of work and are stored in css files.
  • Multiple style definitions will cascade into one
  • Allows developers to control the style and layout of multiple Web pages all at once

Styles Solve a Common Problem

HTML tags designed to define content of document. Layout was supposed to be taken care of by the browser, without using any formatting tags.

As browsers added new tags and attributes to the original HTML spec, it became more and more difficult to separate content of HTML docs with the presentation layout.

W3C stepped in as the responsible party for standardizing HTML and styles. Enter Cascading Style Sheets.

Cascading Order

More than one style can be specified per HTML element. Below is the Order of "cascading" presidence. The higher the number, the higher the priority.

  1. Browser default
  2. External style sheet
  3. Internal style sheet
  4. Inline style (inside an HTML element)
